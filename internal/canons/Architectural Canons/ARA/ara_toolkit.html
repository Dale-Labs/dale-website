Formal Definition
V_comp = V_proj \ V_fixed

Coordination logic
Two Canonical Anchor Modes

ARA resolves the active completion field through a declared coordination structure. The anchor functions as the stabilization reference of Stage 2. Only two anchor modes are admissible. Coordination is not freeform optimization, facilitator negotiation, heuristic ranking, or discretionary balancing.

Zero-Anchor A(v) = 0 across completion field
Completion balancing occurs relative to neutral structural equilibrium conditions inside the inherited metric space. Preserves: uniformity, metric continuity, and execution simplicity across the unresolved architectural region.
Family-Anchor Inherited from operational family structures
Balancing occurs relative to inherited subsystem-family conditions. Preserves: subsystem continuity, family-level reconstruction dependence, and operational-family balancing behavior. Anchor values must remain inherited, recoverable, and traceable.
?Invalid anchor states: hidden anchor substitution (breaks recoverability), runtime anchor improvisation (breaks determinism), undeclared proxy anchors (breaks architectural continuity). Anchor values may not be improvised, inferred heuristically, or substituted by hidden implementation defaults.
Unique solvability
Determinism Requirement

ARA requires unique solvability as a structural validity condition. The completed Stage 2 state must be uniquely generated from: inherited architecture, inherited assigned constants, the declared anchor regime, carry-forward structural conditions, and the declared coordination structure.

Output contract
Canonical Stage 2 Output Triple

The canonical Stage 2 output is the triple (E_comp, E_read, P_S2). All three components are mandatory. Execution becomes invalid if any component is omitted, collapsed, or replaced by simplified substitutes.

E_comp
Completed Value Field
Contains all values resolved through Stage 2 completion. Preserves completion outputs, resolved variable states, metric continuity, and assignment-state lineage. Does NOT overwrite ECOA-assigned constants � fixed values remain inherited while unresolved become resolved.
E_read
Readable Stage 2 Configuration
Reconstructed operational state of the completed architecture. Preserves operational readability, subsystem continuity, family organization, and reconstruction dependence inherited from ECOA.
P_S2
Stage 2 Design Package
Complete DALE-facing completion package for runtime realization, downstream execution, serialization, and traceable continuation into Runner. Runner begins from this package � not from raw completion values alone.

!All Stage 2 outputs must remain reconstructible through the same DALE-grounded architecture inherited from Stage 1. Readable interpretation without recoverability is structurally invalid.
